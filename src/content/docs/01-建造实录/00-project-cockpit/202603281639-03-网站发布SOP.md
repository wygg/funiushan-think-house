# 🚀 Momo 伏牛山居：Learning in Public 自动化发布 SOP

> **核心哲学**：保护私密书写的安全感，同时保持公共发声的穿透力。这套 SOP 通过 Python 脚本建立了坚固的“物理防波堤”，让所有的日记先在 `400-projects` 本地自然流淌，随后一键封装推向浩瀚的 Vercel 云端。

当 Momo 在现场打完水泥、或者你和她进行了一场深度复盘后，如何让全世界看到这段记录？只需严格执行以下 **“铁人三项”命令**：

---

## ✏️ 第一步：私密创作 (在 400-projects 内)
你和 Momo 完全不需要管什么代码或排版，**直接在当前的体系内正常写 Markdown**（无论是存在 `03-mep-engineering/02-execution` 还是 `05-diy-concrete-wall` 里）。
*   想加图片就加图片，想写日记就写日记。
*   只要你们觉得“这篇文章可以公开了”，就进入下一步。

## ⚙️ 第二步：运行“炼丹炉”提纯脚本 (Sync)
打开终端 (Terminal)，将私人笔记一键搬运并自动打上网站所需的数据标签（YAML Frontmatter）：

```bash
# 1. 切换到公共网站专区
cd /Users/wygg/mycode/myAntigravity/930-public-buffer/momo-mountain-house-website

# 2. 运行强大的自动抓取与转换脚本
python3 sync_to_site.py
```
*(看到终端只要弹出 `✅ 所有文档已成功打上...并同步完毕！` 的绿色提示，说明所有的修改已经被物理隔离地推入了网站的前端代码内。)*

## 🚀 第三步：发射核按钮 (Push 到 GitHub)
依然在刚才的终端页面里（`momo-mountain-house-website` 目录下），敲下这极其固定且神圣的三行命令：

```bash
git add .
git commit -m "update: 更新了Momo最新的造屋日记与实学记录"
git push origin main
```

🎉 **至此，人类操作全部结束！** 🎉

---

## ☁️ 幕后黑盒 (发生了什么？)
在你敲下 `git push` 回车键的第 0.1 秒内：
1. 代码被发射到了 GitHub 仓库。
2. GitHub 立刻触发 Webhook 警报给 Vercel。
3. Vercel 的云端集群开始高速运转，自动下载并全网分发（Edge CDN）。
4. **两分钟后**，哪怕是位于纽约或伦敦的城市白领，输入你们的山居专属域名，就能看到最新挂在网页上的那层带灰的清水混凝土记录！
